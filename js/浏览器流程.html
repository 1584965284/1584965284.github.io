输入url 服务器返回一个html 然后浏览器解析html

如果这个html中有link css文件 则会下载css文件

css的下载不会阻塞html的继续解析 会单独下载 不会影响dom解析/dom tree构建 但是会阻塞render tree 因为需要css下载完才行

遇到script标签/js代码/js文件下载 会停止对html的解析 不能继续构建dom树 （但是前面已经构建的dom可以先渲染到界面上 这是浏览器的优化）js文件下载会等待他下载 并且执行完之后才会继续html解析

# defer async

defer async 不会阻塞dom树 defer DOMContentLoaded总是会等待defer中的代码先执行完成 (defer在dom树构建完毕之后调用 DOMContentLoaded事件之前 async下载完之后就立即调用)

另外多个带defer的脚本是可以保持正确的顺序执行的 defer可以提高页面的性能，并且推荐放到head元素中

defer仅适用于外部脚本，对于script默认内容会被忽略 不要在defer标签里写代码 只用来引入外部文件

# render Tree layout paint

css => styleSheets(document.styleSheets)

当有了DOM Tree和 CSSOM Tree(styleSheets)后，就可以两个结合来构建Render Tree了

dom tree与render tree不一定一一对应 比如dom tree有一个display none 那么渲染树就没有

第四步是在渲染树（Render Tree）上运行布局（Layout）以计算每个节点的几何信息

还有分层layer（z-index transform3D video、canvas、iframe）可以提高性能，但是以内存管理为代价，不应过分使用。

paint 浏览器将布局阶段计算的每个frame转为屏幕上实际的像素点 包括将元素的可见部分进行绘制，比如文本、颜色、边框、阴影、替换元素（比如img）

# 合成

浏览器将绘制列表给渲染进程中的合成线程
